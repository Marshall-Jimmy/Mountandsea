# 山海经 · Mountandsea

> **一款基于《山海经》神话体系的 2D 开放世界 RPG。**
> 从随机降生的一片蛮荒开始，采集、狩猎、祭祀、修仙、炼丹、农耕、锻造——在先秦神话世界中走出你自己的文明之路。

**作者**：杨泽宇  
**项目启动**：2022 年 2 月 17 日  
**仓库**：[Marshall-Jimmy/Mountandsea](https://github.com/Marshall-Jimmy/Mountandsea)

## 这是什么游戏？

《山海经》是一部先秦地理志怪典籍，记载了上百座山川、数百种异兽草木、数十个方国神话。本项目试图将这部两千多年前的文本转化为一款可玩的开放世界 RPG：

- 你会**随机降生**在世界中的某个村落，出生地决定你的初始资源和周边环境
- 你可以**采集祝余充饥、狩猎狌狌增速、佩戴迷榖防迷路**——山海经原文中的每一样东西都有游戏用途
- 你可以**祭祀山神获得庇佑**，祭品规则严格遵循原文中各山经的「祠之礼」
- 你可以**修仙引气、筑基、御物、御兽**，最终飞升成仙
- 你可以**发展中世纪西医外科手术**，基于四液理论放血灌肠，科技达标后推翻旧理论
- 你**死亡后可以控制自己的后代继续游戏**，尸体还能用作肥料——血脉延续，生生不息
- 剧情任务**没有奖励**，只做背景叙事和 NPC 铺垫，强调沉浸式体验

## 项目特点

### 1. 六大科技树——从原始到文明

游戏的核心进度系统是一棵**六分支科技树**，玩家可自由选择发展路线，跨分支组合解锁高级科技：

| 分支 | 从 | 到 | 亮点 |
|------|----|----|------|
| 生存 | 采集 | 驯兽 | 狩猎异兽、生火烹饪、建造住所、驯服鹿蜀九尾狐 |
| 医药 | 草药学 | 炼丹 | 解毒术、西医四液外科手术（可推翻）、不死药 |
| 修仙 | 引气 | 飞升 | 筑基、御剑飞行、御兽、飞升成仙 |
| 祭祀 | 基础祭祀 | 祈愿术 | 山神祭祀（严格遵循原文）、通灵术、风调雨顺祈愿 |
| 工匠 | 石器 | 机械 | 金属冶炼、玉器加工、弩机、奇肱国飞车 |
| 农牧 | 种植 | 百谷 | 灌溉水车、畜牧牛羊、育种改良、后稷百谷 |

> 详见 [科技树文档](docs/山海经附录2-科技树.md)

### 2. 死亡与传承机制

玩家死亡后可以选择**控制自己的后代**继续游戏，实现血脉延续。任何生物的尸体都可以用作**肥料**。佩戴鹿蜀角可微弱增强后代天赋——这源自原文「佩之宜子孙」的设定。

### 3. 西医中世纪外科手术

这是游戏最具特色的创意之一。医疗科技发展到一定阶段后，玩家可以解锁基于**四体液学说**（血、黏液、黄胆汁、黑胆汁）的外科手术系统：

- **阶段一**：放血疗法、灌肠术、草药敷料（理论源头：希波克拉底、盖伦）
- **阶段二**：科技达标后**推翻四液理论**，进入解剖学、消毒术、缝合术

这一设计体现了「科技推翻旧理论」的核心理念。创意来源：手游《神医》、纪录片《无影灯下》。

### 4. 祭祀系统——严格遵循原文

祭祀系统的祭品规则**逐字遵循山海经原文**。每个山系的山神有不同的状貌和祭品要求：

| 山系 | 山神状貌 | 祭品 |
|------|---------|------|
| 南山经 | 鸟身龙首 | 毛用一璋玉瘗，糈用稌米，一璧，稻米白菅为席 |
| 北山经 | 人面蛇身 | 毛用一雄鸡彘瘗，吉玉用一圭 |
| 西山经 | — | 太牢（华山冢）、百牺百瑜（羭山神） |

### 5. 文言文语言选项

游戏内提供**文言文语言选项**，玩家可以选择以先秦文言文进行游戏。创意来源：Minecraft《山海经》模组。

### 6. 剧情任务无奖励

剧情任务**没有物质奖励**，仅作为背景介绍、故事引导或 NPC 背景铺垫。这是对「任务=奖励」这一 RPG 套路的反叛，强调沉浸式叙事体验。

## 技术选型

项目已完成 2D 开放世界游戏引擎架构选型调研，对 8 个开源仓库进行了代码级审计。核心结论：

| 需求场景 | 推荐方案 | 理由 |
|---------|---------|------|
| Terraria 式挖掘编辑 + 液体 + 物理 | Substrata-Game 架构蓝本 | WorkerThreadPool 多线程 + 自绘 shader + 自定义物理 + 可编辑地形，架构最完整 |
| 俯视角程序化生成 | Gaea 插件 | 节点图 + 多线程 + chunk 流式，MIT 协议，活跃维护 |
| 最快出原型 | Gaea 插件 | 安装即用，可视化节点图编辑器 |
| 长期可维护 | Substrata + Gaea | 自绘渲染绕过 API 变更风险 + 插件独立升级 |

> 详见 [选型调研报告](docs/2d-openworld-survey.md)
